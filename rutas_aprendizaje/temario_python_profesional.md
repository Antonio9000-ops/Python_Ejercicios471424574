# Temario de Python profesional

Este temario organiza los fundamentos y prácticas necesarias para avanzar desde una base de Python hacia un perfil profesional, con énfasis en herramientas reales de trabajo, calidad de código y preparación laboral.

## 1. Fundamentos para resolver problemas

- Pensamiento computacional: descomposición, reconocimiento de patrones, abstracción y diseño de algoritmos.
- Complejidad algorítmica básica: Big O, coste temporal, coste espacial y comparación práctica entre soluciones.
- Estructuras de datos prácticas: listas, tuplas, diccionarios, conjuntos, pilas, colas, árboles básicos y cuándo usar cada una.
- Manejo de errores realista: excepciones esperadas, errores recuperables, validaciones, mensajes útiles y fallos seguros.

## 2. Herramientas profesionales desde el inicio

- Git desde el inicio: repositorios, ramas, historial, `add`, `commit`, `status`, `diff`, `log`, `pull`, `push` y resolución básica de conflictos.
- Buenas prácticas de commits: mensajes claros, cambios pequeños, commits atómicos y revisión antes de confirmar.
- Uso profesional de terminal: navegación, gestión de archivos, variables, pipes, redirecciones, permisos y comandos frecuentes.
- Entornos virtuales: creación, activación, aislamiento de dependencias y uso de `requirements.txt` o herramientas equivalentes.
- Configuración con variables de entorno: separación entre código y configuración, secretos, `.env` y valores por entorno.

## 3. Lectura, estilo y calidad de código

- Lectura de documentación: búsqueda eficiente, lectura de APIs, ejemplos oficiales, docstrings y evaluación de fuentes.
- Type hints en Python: anotaciones básicas, colecciones tipadas, `Optional`, `Union`, tipos propios y beneficios para mantenimiento.
- Formateadores y linters: Black y Ruff para consistencia, detección de errores comunes e integración en el flujo diario.
- Logging: niveles de log, mensajes accionables, trazabilidad, reemplazo de `print` en aplicaciones y configuración básica.

## 4. Testing y confiabilidad

- Testing desde nivel intermedio: pruebas unitarias, pruebas de integración, fixtures, parametrización y organización de casos de prueba.
- Diseño de pruebas para casos normales, bordes y errores.
- Automatización de checks antes de entregar código: formato, linting y ejecución de tests.

## 5. Datos, APIs y backend

- SQL obligatorio: consultas `SELECT`, filtros, joins, agregaciones, índices básicos, modelado relacional y buenas prácticas.
- Diseño de APIs: recursos, contratos, códigos HTTP, validación de entradas, paginación, versionado y documentación.
- Seguridad básica: manejo de secretos, validación de datos, autenticación básica, permisos, dependencias seguras y prevención de errores comunes.

## 6. Preparación profesional

- Preparación para entrevistas técnicas: resolución de problemas, explicación de decisiones, análisis de complejidad, ejercicios con estructuras de datos y práctica de comunicación.
- Construcción de proyectos demostrables con README, instrucciones de ejecución, tests y commits comprensibles.
- Revisión de código propia y de terceros para mejorar claridad, mantenibilidad y colaboración.
